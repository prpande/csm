# Plans

Implementation plans for **standard-tier** issues (see
[`../workflow/issue-driven-development.md`](../workflow/issue-driven-development.md)
§2). A plan is a short doc — what/why, the approach, and the test list — written
before coding. High-risk work gets a fuller spec in [`../specs/`](../specs/)
instead; trivial hygiene work needs neither.

Name plans `<issue#>-<slug>.md`.
